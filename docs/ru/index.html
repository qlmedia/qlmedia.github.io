QLMedia

Расширение QuickLook macOS для предварительного просмотра видео/аудио.

Проблемы / информация

(сделать: сортировку по разделам, дополнить, добавить снимки экранов)

  • Общая информация:
    • Минимальная версия macOS: 10.15 (Catalina), в более ранних версиях отсутствует требуемый функционал.
    • Для активации расширения требуется запустить QLMedia.app как минимум один раз.
    • Ссылка на приложение в App Store: https://apps.apple.com/ru/app/qlmedia/id1667620638
    • Интерфейс тормозит по причине его работы в изолированном окружении QuickLook.
    • QLMedia.app регистрирует в системе тип файла .vlcf. Для таких файлов будет попытка сделать эскиз и отобразить его предварительный просмотр. Для их открытия в нужном приложении - настройте его в Finder.
  • Воспроизведение:
    • При включенном режиме "Пауза на последнем кадре" предполагается, что после паузы в случае необходимости дальнейшего просмотра сначала будет изменена текущая позиция. Если будет сразу запущено воспроизведение без предварительного изменения текущей позиции, то воспроизведение будет завершено.
  • Многоэкранность:
    • macOS с несколькими экранами работает плохо – теряет порядок их подключения, моргают экраны, сбрасывается разрешение/HDR и т.д.
      Для лучшего контроля за подключёнными экранами можно использовать например:
      • displayplacer
      • SwitchResX
      • Stay
    • При отключении(/подключении?) экранов бывает "залипание" – превью перестаёт реагировать на клики.
      Исправляется рестартом Finder
      • через системное меню
      • запуском скрипта Reset Finder из QLMedia.app
      • из командной строки: $ pkill -1 Finder
      Вызов qlmanage -p <file> работает корректно.
    • При работе с 3+ экранами предпочитаемый экран для полноэкранного просмотра выставляется нажатием на кнопку в QLMedia.app / Experimental / Play on screen (кнопка) после переноса приложения на требуемый экран.
      На последних macOS можно использовать степпер.
  • Проблемы:
    • MacOS Sequoia: Чтобы генератор иконок работал корректно, пожалуйста включите «Use default Icon Flavor» в разделе «Experimental».
    • Не работает на .avi/.mpeg/... т.к. macOS не даёт доступа к превью подобных файлов.
    • В режиме галереи переход к следующему файлу не приводит к завершению проигрывания текущего.
